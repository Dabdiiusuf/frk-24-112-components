# README — Building Button Atoms from `ExampleButton.molecule`

This is a **dead-simple guide** for turning one reusable **molecule** (`ExampleButton`) into many tiny **atoms** (`Start`, `PlayAgain`, `Timer`, etc.).

---

## 1. What the Molecule Does

`ExampleButton` is a generic `<button>` that can be:

- **Text-only** (normal button), or
- **Image-skinned** (shows PNGs as background for idle/hover/active states).

**Props:**

- `onClick` — function to call on click
- `children` — what to render inside (text or an invisible label)
- `className` — add extra classes from outside
- `imageSet` — `{ idle, hover, active }` image URLs _(optional)_
- `width`, `height` — size (`number` = px, or a CSS string like `"10rem"`)

If `imageSet` is provided, the molecule applies a background image using **CSS variables**.

---

## 2. Build a Text Atom (No Images)

```jsx filename="PrimaryTextButton.atom.jsx"
import React from "react";
import Button from "./ExampleButton.molecule";

const PrimaryTextButton = ({ onClick, label = "Click me" }) => (
  <Button onClick={onClick} className="myExtraClass">
    {label}
  </Button>
);

export default PrimaryTextButton;
```

## 3. Build an Image-Skinned Atom

```jsx
import React from "react";
import Button from "./ExampleButton.molecule";
import idle from "./images/start-idle.png";
import hover from "./images/start-hover.png";
import active from "./images/start-active.png";
import styles from "./ExampleButton.module.css"; // for .srOnly (optional)

const StartButton = ({ onClick, width = 320, height = 110 }) => (
  <Button
    imageSet={{ idle, hover, active }}
    width={width}
    height={height}
    onClick={onClick}
  >
    {/* Visible text optional; this keeps it accessible */}
    <span className={styles.srOnly}>Start</span>
  </Button>
);

export default StartButton;
```

## 4. A Logic Atom that Switches Skins (Start ↔ Play Again)

```jsx
import React from "react";
import Button from "./ExampleButton.molecule";
import startIdle from "./images/start-idle.png";
import startHover from "./images/start-hover.png";
import startActive from "./images/start-active.png";
import againIdle from "./images/again-idle.png";
import againHover from "./images/again-hover.png";
import againActive from "./images/again-active.png";

const StartPlayAgain = ({
  started = false,
  onStart,
  onPlayAgain,
  width = 320,
  height = 110,
}) => {
  const imageSet = started
    ? { idle: againIdle, hover: againHover, active: againActive }
    : { idle: startIdle, hover: startHover, active: startActive };

  const handle = started ? onPlayAgain : onStart;

  return (
    <Button imageSet={imageSet} width={width} height={height} onClick={handle}>
      <span className="sr-only">{started ? "Play again" : "Start"}</span>
    </Button>
  );
};

export default StartPlayAgain;
```

## 5. Show It in Storybook

```jsx
import React from "react";
import StartPlayAgain from "./StartPlayAgain.atom";

export default { title: "Components/Button/StartPlayAgain" };

export const Interactive = {
  render: () => {
    const [started, setStarted] = React.useState(false);
    return (
      <StartPlayAgain
        started={started}
        onStart={() => setStarted(true)}
        onPlayAgain={() => setStarted(false)}
      />
    );
  },
};
```

## 6. Mental Model (TL;DR)

- The **molecule** is a plain, reusable button.
  It knows how to paint an image skin but **not which one**.

- Each **atom** supplies:
  - which images to use (`imageSet`)
  - what size it is (`width`, `height`)
  - what it does (`onClick`)
  - optional label via `children` for accessibility

### Build as many atoms as you like by changing those four things — **no need to touch the molecule**.
